La idea

De una idea a un proyecto que funciona — y que crece.

No hace falta ser programador. Vas a describir en tu idioma lo que querés, un agente de IA lo construye, lo publicás con una URL real, le sumás datos, usuarios y emails, y lo enriquecés con información del mundo. Estas son las herramientas que usamos en clase.

El mapa

Cinco movimientos, unas pocas herramientas.

1 · Construir
Agentes de código
Codex · Claude Code · Antigravity. Les decís qué querés; escriben el código.
2 · Publicar
GitHub + Vercel
Guardás el proyecto y lo ponés online con una dirección web real.
3 · Datos y usuarios
Supabase
Base de datos, login de usuarios y panel de administración.
4 · Comunicar
Resend
Que tu web mande emails automáticos (con una API key).
5 · Enriquecer
Apollo · Enrichlayer · Perplexity
Sumar información del mundo real para que tu proyecto valga más.
La lógica
Construir → enriquecer
Primero algo que funcione. Después, cada capa lo hace más útil.
Parte 1

Construir

De la idea a una web publicada, con datos, usuarios y emails.

1
Paso 1 · Construir

El agente de código escribe por vos.

Le describís lo que querés en lenguaje natural — "una web para inscribir alumnos a un taller, con un formulario y una lista" — y arma el proyecto. Tres opciones equivalentes:

⌨️
Claude Code
Agente de Anthropic en la terminal. Crea y edita proyectos enteros conversando.
🧩
Codex
Agente de código de OpenAI. Escribe, corrige y explica el código.
🚀
Antigravity
Entorno de desarrollo agéntico de Google. El agente trabaja el proyecto con vos.

Cómo empezar: describí el proyecto en una frase, pedile que lo cree, y luego pedí cambios de a uno ("agregá un botón para exportar", "cambiá el color del título"). Se avanza conversando.

Paso 2 · Publicar

GitHub guarda. Vercel publica.

🐙
GitHub
Es la "carpeta en la nube" del proyecto: guarda cada versión y el historial de cambios. Nada se pierde.
▲
Vercel
Toma tu código de GitHub y lo pone online con una URL real. Cada cambio que subís se publica solo.

Paso 3 · Datos y usuarios

Supabase: memoria y puerta de entrada.

Hasta acá tu web muestra cosas, pero no recuerda nada. Supabase le da una base de datos y un sistema de usuarios.

🗄️
Base de datos
Guardás y consultás información: inscriptos, pedidos, registros. Lo que tu proyecto necesite recordar.
🔑
Usuarios / login
Que la gente inicie sesión. Cada uno ve lo suyo; vos controlás quién entra.
🛠️
Panel admin
Una pantalla interna (web/admin) para gestionar los datos sin tocar código.
Paso 4 · Comunicar

Que tu proyecto escriba emails solo.

Resend es el servicio que le permite a tu web enviar correos: una confirmación de inscripción, un aviso, un recordatorio. Se conecta con una API key — una llave que le decís al agente que use, y listo.

Formularioalguien se anota
→
Tu webguarda en Supabase
→
Resendmanda el email

Qué es una API key: una contraseña que conecta dos servicios. Nunca se comparte ni se sube público — se guarda en la configuración del proyecto (en Vercel), y el agente la usa por vos.

Paso 5 · Sumar inteligencia

Un asistente adentro de tu web.

Con la misma idea, le pedís al agente que agregue un chat o asistente de IA a tu proyecto: responde preguntas, guía al usuario, o completa tareas — usando la información de tu propia base de datos.

💬
Atiende
Responde consultas frecuentes las 24 horas.
🧭
Guía
Ayuda al usuario a hacer lo que vino a hacer.
⚡
Actúa
Busca, resume o completa formularios por el usuario.
Parte 2

Enriquecer

Tu proyecto ya funciona. Ahora le sumamos información del mundo real.

2
La diferencia

Un proyecto vale por los datos que maneja.

Dos webs iguales por fuera no valen lo mismo: la que sabe más del mundo — quién es el cliente, qué pasa en su industria, qué información hay disponible — sirve mucho más. Enriquecer es traer ese conocimiento a tu proyecto, automáticamente.

Paso 6 · Investigar

Perplexity: respuestas con fuentes.

Es un buscador con IA que responde preguntas y cita de dónde salió cada dato. Tu proyecto puede consultarlo para investigar en el momento: un mercado, una empresa, una tendencia — con información fresca y verificable.

Ejemplo: un usuario carga el nombre de una empresa y tu web, por detrás, le pide a Perplexity un resumen actualizado del sector — sin que nadie lo busque a mano.

Paso 7 · Datos de empresas y contactos

Apollo: quién es quién en el mundo B2B.

Apollo es una base enorme de empresas y contactos profesionales. Tu proyecto puede consultarla para completar datos de un cliente: rubro, tamaño, ubicación, personas de contacto — al instante.

🏢
Empresas
Rubro, tamaño, país, sitio web.
👤
Contactos
Personas y sus roles dentro de la empresa.
🎯
Segmentar
Encontrar a quién apuntar, con criterios claros.
Paso 8 · Completar el dato

Enrichlayer: de un dato suelto a un perfil completo.

Muchas veces tenés solo una pieza — un email, un nombre, un link. Enrichlayer lo completa: trae el perfil profesional, la empresa, el cargo. Ideal para que tu formulario pida poco y tu base sepa mucho.
